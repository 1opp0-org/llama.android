# Summary
- **Status**: New Feature.
- **What it's about**: Using GBNF (GGML BNF) grammars to force Large Language Model (LLM) output into specific formats, such as JSON, that can directly map to Kotlin data classes. This ensures the model's output adheres to a predefined structure, making it reliable for programmatic use.
- **JNI Implementation**: This feature requires JNI-level code to bridge a Kotlin API that converts data classes into JSON schemas. This schema is then passed to the underlying C++ `llama.cpp` library, which uses its internal `json-schema-to-grammar` converter to enforce the output format during token generation. No significant changes are expected in the core `llama.cpp` codebase itself.
- **Estimation**: 
    - **C++ JNI**: Approximately 30-50 lines of code for passing the grammar string and handling potential errors.
    - **Kotlin**: Approximately 100-150 lines of code for generating the JSON schema from a data class (e.g., using `kotlinx-serialization` or reflection) and handling the serialization/deserialization.
- **Caveats**: 
    - Large or complex JSON schemas can potentially introduce a slight overhead in token generation speed, as more validation is performed at each step.
    - A robust and efficient method for converting Kotlin data classes to JSON schemas needs to be established and maintained.
    - Error handling for invalid schemas passed from Kotlin to C++ should be carefully implemented.
